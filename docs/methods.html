Methods Explained — Node.js CLI Projects

ဤစာမျက်နှာတွင် Expense Tracker နှင့် GitHub Activity CLI တွင် သုံးထားသည့် method/function များကို မြန်မာလို တစ်ခုပြီးတစ်ခုရှင်းပြထားပါတယ်။

process.argv

Why: CLI ကနေ user ထည့်သွင်းတဲ့ command နဲ့ flags ကို ရယူရန်။
How: Node က program path, script path, args စာရင်းကို array အဖြစ်ပေးတယ်။ slice(2) နဲ့ user args ကိုပဲ ယူတယ်။

const args = process.argv.slice(2); // ['add','--description','Lunch','--amount','20']

process.cwd()

Why: Data file ကို run လုပ်နေတဲ့ working directory အပေါ် မူတည်ချင်လို့။
How: လက်ရှိ working directory ကို absolute path အဖြစ် ပြန်ပေးတယ်။

const cwd = process.cwd(); // e.g., C:\Users\user\node-js-training

path.resolve()

Why: OS အလိုက် path ပေါင်းစပ်မှားမှု မဖြစ်အောင် absolute path တည်ဆောက်ရန်။
How: segments တွေကို resolve လုပ်ပြီး ./.. ကို normalize ပြုလုပ်တာ၊ absolute မတွေ့ရင် process.cwd() ကို အခြေခံတယ်။

import path from 'path';
const file = path.resolve(process.cwd(), 'expenses.json');

fs.existsSync / fs.readFileSync / fs.writeFileSync

Why: အလွယ်တကူ synchronous နည်းလမ်းနဲ့ ဖိုင် စစ်/ဖတ်/ရေး လုပ်ရန် (CLI လေးများအတွက် ရိုးရိုးပဲ လိုတယ်)။
How: မရှိရင် [] နဲ့ ဖန်တီး၊ ဖတ်ရာတွင် UTF-8, ရေးရာတွင် JSON.stringify(..., 2) သုံးပေးတယ်။

import fs from 'fs';
if (!fs.existsSync(file)) fs.writeFileSync(file, '[]', 'utf8');
const text = fs.readFileSync(file, 'utf8');
fs.writeFileSync(file, JSON.stringify(data, null, 2), 'utf8');

Browser မှာ မရပါ — Node.js-only API ဖြစ်တယ်။

JSON.parse / JSON.stringify

Why: Data ကို JSON text ↔ JS object/array ပြောင်းရန်။
How: JSON.parse သည် error ပစ်နိုင်လို့ try/catch သုံး၊ JSON.stringify(value, null, 2) သည် human-readable ဖြစ်စေတယ်။

const arr = JSON.parse('[{"id":1,"amount":20}]');
const text = JSON.stringify(arr, null, 2);

Array Methods

Why: List ကို CRUD လုပ်ရန်၊ totals တွက်ရန်။
How: findIndex (ရှာ index), push (ထည့်), splice (ဖျက်), reduce (စုစုပေါင်း), filter (စစ်), slice (copy အစိတ်အပိုင်း)။

const i = arr.findIndex(e => e.id === id);
arr.push(newItem);
arr.splice(i, 1);
const total = arr.reduce((s,e)=>s+Number(e.amount||0), 0);
const monthList = arr.filter(e => String(e.date).startsWith('2025-09'));
const first30 = arr.slice(0, 30);

Number & String Methods

Why: Validation (id/amount/month) နှင့် output formatting အတွက်။
How: Number(), Number.isFinite, Number.isInteger, toFixed(2), trim, padEnd, startsWith။

const amount = Number('12.5');
Number.isFinite(amount) && amount > 0;
Number.isInteger(Number('3'));
amount.toFixed(2); // '12.50'
'Lunch'.trim();
'Lunch'.padEnd(25, ' ');
'2025-09-12'.startsWith('2025-09');

Date Formatting (YYYY-MM-DD)

Why: Monthly summary အတွက် filter လွယ်ကူစေခြင်း။
How: Year/Month/Day ကို zero-pad နှင့် template literal နဲ့ ပေါင်းတယ်။

function todayIsoDate(){
  const d = new Date();
  const y = d.getFullYear();
  const m = String(d.getMonth()+1).padStart(2,'0');
  const day = String(d.getDate()).padStart(2,'0');
  return `${y}-${m}-${day}`;
}

Optional chaining (?.) / Nullish coalescing (??)

Why: External data (e.g., GitHub API) မှာ field မရှိနိုင်တာကြောင့် crash မဖြစ်စေဖို့။
How: a?.b?.c က safe access, x ?? fallback က null/undefined ဖြစ်မှ fallback သတ်မှတ်တယ်။

const repo = ev?.repo?.name ?? '(unknown repo)';

Template Literals

Why: Dynamic messages ကို ရှင်းလင်းစွာ သရုပ်ဖော်ရန်။
How: backticks (`) နဲ့ ${expr} interpolation သုံးတယ်။

console.log(`Expense added successfully (ID: ${id})`);

encodeURIComponent (for URLs)

Why: GitHub API path ထဲ ထည့်မယ့် username ကို URL-safe လုပ်ရန်။
How: unsafe chars ကို percent-encode လုပ်တယ်။

const path = `/users/${encodeURIComponent(username)}/events`;

https (Node.js)

Why: External APIs (GitHub) ကို library မသုံးဘဲ built-in နဲ့ခေါ်ရန်။
How: https.request(options, cb) → collect chunks → JSON.parse → handle status codes (403/404...)။

import https from 'https';
const req = https.request({ hostname:'api.github.com', path:'/users/octocat/events', method:'GET', headers:{'User-Agent':'cli'} }, res => {
  let data='';
  res.on('data', c => data += c);
  res.on('end', () => { const json = JSON.parse(data || '[]'); console.log(json.length); });
});
req.end();

process.exit / process.exitCode

Why: Error ဖြစ်တဲ့အခါ exit status ကို မှန်ကန်စွာ ပေးရန်၊ သို့မဟုတ် ချက်ချင်းပိတ်ရန်။
How: exitCode သတ်မှတ်ရုံက graceful; exit() က immediate (async အချို့ မပြီးနိုင်)။

process.exitCode = 1; // graceful error exit
// process.exit(1);   // immediate exit
